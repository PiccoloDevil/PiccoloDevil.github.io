本篇详细介绍了List实现类中的ArrayList、Vector和LinkedList

ArrayList

ArrayList是容量可变的动态数组

特点

底层实现：数组Object[]

查找快，增删慢，线程不安全

常用方法

ArrayList实现了List接口，常用方法示例参照List接口的demo

  • add(E e) 添加元素
  • remove(int index) 删除指定索引处的元素
  • clear() 清空列表
  • clone() 返回ArrayList实例的浅拷贝
  • size() 返回元素个数
  • get(int index) 通过索引获取元素
  • indexOf(Object o) 返回指定元素索引值
  • contains(Object o) 判断是否包含
  • isEmpty() 判断是否为空
  • subList(int fromIndex, int toIndex) 获取子列表
  • set(int index, E element) 用指定的元素替换指定位置的元素

源码分析

快速随机访问：通过元素的序号快速获取元素对象(对应于get(int index)方法)

扩容机制

为什么扩容因子是1.5？

  • 扩容的容量太小会导致需要频繁扩容，从而需要频繁的申请内存空间和数组拷贝，从而降低性能

  • 扩容的容量太大会导致内存空间资源浪费

扩容因子为1.5时，就能充分利用空间资源，且1.5的设置可以充分利用移位操作，减少了浮点数的使用，减少了运算次数和运算时间，从而增加了效率

Vector

Vector也是动态数组，但是在现在的开发中已经不常用了

特点

底层实现：数组Object[]

查找快、增删慢，线程安全

由于是线程安全的，区别于ArrayList，故运行效率比ArrayList要稍慢一点

这也是为什么Vector不被推荐使用的原因：作为普通的集合类，每个方法都实现了同步，极大地影响了使用效率

常用方法

由于Vector也实现了List接口，故一些常用方法的使用类似，不再赘述

Vector从JDK1.0开始就有了，因此它还包含了许多传统的方法

枚举器与迭代器之间的区别

枚举是初始Java版本JDK1.0的一部分，尽管迭代器与集合框架一起包含在JDK 1.2中，该框架也仅在JDK 1.2中添加。迭代器被设计为仅专注于集合框架，在Java集合框架中，迭代器代替了枚举

迭代器与枚举有两种不同：

  1. 迭代器允许调用者在迭代过程中使用定义明确的语义从基础集合中删除元素
  2. 方法名称已得到改进
枚举器 迭代器
hasMoreElement() hasNext()
nextElement() next()
不适用 remove()

Java API 规范建议，对于较新的程序，应首选迭代器而不是枚举

总结

ArrayList与Vector的区别：ArrayList线程不安全，Vector线程安全

ArrayList与LinkedList的区别：

  1. 底层实现：ArrayList数组实现；LinkedList双向链表实现
  2. 适用场景：ArrayList适用于频繁的查找；LinkedList适用于频繁的增加和删除
  3. 空间连续性：ArrayList必须开辟连续空间；LinkedList无须开辟连续空间
  4. 快速随机访问性：ArrayList实现了RandomAccess接口，支持快速随机访问；LinkedList没有实现RandomAccess接口，不支持快速随机访问
