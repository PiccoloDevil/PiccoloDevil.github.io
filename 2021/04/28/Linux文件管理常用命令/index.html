学习记录一下Linux中文件与目录管理相关的命令操作

cd

说明：进入某个目录

用法：

  • cd ./ 进入当前目录
  • cd ../ 进入当前目录的上一级目录

alias

说明：设置指令别名

用法：alias [别名] = [指令名称]

补充：

为什么使用命令时直接输入命令名，不用添加其绝对路径？

先打印出环境变量PATH的值查看一下

再查看下ls命令的绝对路径

可以看到，环境变量PATH中已经记录了/usr/bin/这个路径，所以当我们使用ls命令时就能直接输入ls，而不用输入其完整路径了

ln

说明：用于为文件创建连接

用法：ln –s [源文件] [目标文件]

-s 创建软连接（也叫符号连接）

umask

说明：指定在建立文件时预设的权限掩码

Q: 在linux系统中，默认一个目录的权限为 755（drwxr-xr-x），而一个文件的默认权限为644（-rw-r—r—）。这个权限值是怎么规定的呢？

A: umask指定在建立文件时预设的权限掩码

  1. 若用户建立为普通文件，则预设“没有可执行权限”，只有rw两个权限。最大为666（-rw-rw-rw-）
  2. 若用户建立为目录，则预设所有权限均开放，即777（drwxrwxrwx）

umask值的含义：上述两条规则中的默认值需要减掉的权限值

umask默认为0022，还可自定义umask值

umask 可以在/etc/bashrc里面更改，预设情况下，root的umask为022，而一般使用者则为002，因为可写的权限非常重要，因此预设会去掉写权限

mkdir

说明：创建一个目录

若上级目录不存在或重复创建已存在的目录，则会提示报错无法创建，解决方案：可用 -p 参数递归地创建

touch

说明：如果该文件存在，则将其时间标签更新为系统当前的时间，否则创建新的空文件

groupadd

说明：增加一个用户组

rmdir

说明：删除一个目录

类似的，-p 参数会递归删除命令后带的路径上的所有目录

但是rmdir无法删除一个不为空的文件夹，且只能删目录不能删文件

rm

说明：删除目录或文件

当要用rm删除目录时需加上 -r 参数

-f 强制删除（不加 -f 的话，删除一个不存在的文件时会报错）

cp

说明：拷贝文件或目录

用法：cp [选项] [ 源文件 ] [目标文件]

如果需要拷贝目录需加上 -r 参数

cp命令带上 -d 参数时，拷贝的是连接文件（快捷方式），不带 -d 参数拷贝软连接时会把软连接实际连接到的目标文件拷贝过去

-i 询问是否覆盖已存在的文件

-u 该选项仅当目标文件存在时才会生效，如果源文件比目标文件新才会拷贝，否则不做任何动作

案例：

mv

说明：移动文件

用法：mv [ 选项 ] [源文件] [目标文件]

可用于Linux下重命名

案例：

chgrp

说明：更改所属组

用法：chgrp [组名] [文件名]

案例：

chown

说明：更改所属主

用法：

chown [ -R ] 账户名 文件名

chown [ -R ] 账户名:组名 文件名

-R 参数只作用于目录，作用是级联更改，即不仅更改当前目录，连目录里的目录和文件也全部更改

案例：

chmod

说明：更改权限

用法：chmod [-R] xxx 文件名 （这里的xxx表示数字）

补充：

  • -R：级联更改

  • r：4；w：2；x：1；-：0

  • 在linux系统中，默认一个目录的权限为 755（drwxr-xr-x），而一个文件的默认权限为644（-rw-r—r—）

    即默认目录的权限为：所属主拥有全部权限，所属组和其他用户只可打开该目录（目录的执行权限等同于可读权限）

    默认文件的权限为：所属主可读可写，所属组和其他用户只可读

特别的：chmod还支持使用rwx的方式来设置权限

  • u：user（所属主）
  • g：group（所属组）
  • o：others（其他用户）
  • a：all（全体）
  • +：加入权限
  • -：删除权限
  • =：赋值权限

案例：

chattr

说明：修改文件的特殊属性（暂不了解~）

ls

说明：查看某个目录或文件

-a 会列出所有的文件和目录（包括隐藏的）

-l 会详细列出文件的属性信息，包括大小、创建日期、所属主所属组等等

ls -l 等同于 ll

pwd

说明：显示当前所在目录

cat

说明：查看文件内容并显示在屏幕上

-n 参数显示行号

-A 显示所有内容，包括特殊字符

tac

说明：反向cat，反向打印

more

说明：查看文件内容

区别于cat的是，cat出的内容一个屏幕放不下的时候无法看到前面的，more可以按空格继续看下一屏，按 q 提前退出

less

说明：查看文件内容

类似于more，除了按空格外，还能上下翻。按 j 下移一行，按 k 上移一行。特别的，more和less查看文件时，还可以搜索特定字符，按 / 输入word是当前行向下搜，按 ? 输入word是当前行向上搜，搜出多个按 n 显示下一个

head

说明：查看文件内容

默认显示文件的前十行。-n 参数可设置显示行数，如：head -n 5 [文件名]，将显示文件的前5行

tail

说明：查看文件内容

类似于head，tail显示最后十行，同样可用 -n 参数指定行数

如果文件是不断增加的，可用 -f 参数动态显示

lsattr

说明：列出文件/目录的特殊属性（暂不了解~）

which

说明：查找一个命令的绝对路径

which只能用来查找PATH环境变量中出现的路径下的可执行文件

当查找的文件在PATH变量中并没有时，就会报错

whereis

说明：查找二进制程序（-b）、源代码文件（-s）和man手册页（-m）等相关文件路径

和find相比，whereis查找的速度非常快

因为Linux系统会将系统内的所有文件都记录在一个数据库文件中，当使用whereis和locate命令时，会从数据库中查找数据，而不是像find那样通过遍历硬盘来查找，所以效率更高

但是该数据库文件并不是实时更新的，默认情况下一星期更新一次，因此我们在用whereis和locate查找文件时，有时会找到已经被删除的数据，或者刚刚建立文件却无法查找到，原因就是数据库文件没有被更新

locate

说明：类似于whereis，也是通过查找预先生成的文件列表库来告诉用户要查找的文件在哪里

具体用法不深究，用的不多~

find

说明：Linux下最常用的搜索工具，在指定目录下查找文件

用法：find [路径] [参数]

案例：

  • 列出当前目录及子目录下所有文件和文件夹

  • 在/home目录下查找以.txt结尾的文件名（若要忽略大小写则使用-iname）

  • 当前目录及子目录下查找所有以.txt和.pdf结尾的文件

  • 匹配文件路径或者文件

  • 基于正则表达式匹配文件路径

  • 找出/home目录下不是以.txt结尾的文件（否定参数）

  • 根据文件类型进行搜索

    类型参数列表：

    • f：普通文件
    • l：符号连接（软连接）
    • d：目录
    • c：字符设备
    • b：块设备
    • s：套接字
    • p：Fifo

  • 根据文件时间戳进行搜索

    Linux系统每个文件都有三种时间戳：

    • 访问时间（-atime/天，-amin/分钟）：用户最近一次访问时间
    • 修改时间（-mtime/天，-mmin/分钟）：文件最近一次修改时间
    • 变化时间（-ctime/天，-cmin/分钟）：文件数据元（例如权限等）最后一次修改时间

    例如：搜索最近7天内被访问过的所有文件

关于find还有很多种用法，暂且列举这么些~
