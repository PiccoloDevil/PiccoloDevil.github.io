Collections是JDK提供的一个工具类，本篇介绍了Collections类的常用方法

官方文档

This class consists exclusively of static methods that operate on or return collections. It contains polymorphic algorithms that operate on collections, “wrappers”, which return a new collection backed by a specified collection, and a few other odds and ends.

The methods of this class all throw a NullPointerException if the collections or class objects provided to them are null.

翻译：

此类仅包含对集合进行操作或返回集合的静态方法。它包含对集合进行操作的多态算法，“包装器”，该包装器返回由指定集合支持的新集合，以及其他一些零碎的结局。

如果提供给它们的集合或类对象为null，则此类的所有方法都将引发NullPointerException。

Collections是JDK提供的一个工具类，位于java.util包下，它提供的方法都是静态的，能更方便的操作各种集合

下面介绍一些Collections类常用的方法

常用方法

空集合

  • List<T> emptyList() 创建空List

  • Map<K, V> emptyMap() 创建空Map

  • Set<T> emptySet() 创建空Set

注意：这些方法创建的空集合都是不可变集合，无法向其中添加或删除元素

补充：利用集合接口提供的of(T...)方法创建空集合，与此方法等效

单元素集合

  • List<T> singletonList(T o) 创建单元素List

  • Map<K, V> singletonMap(K key, V value) 创建单元素Map

  • Set<T> singleton(T o) 创建单元素Set

注意：这些方法创建的单元素集合也都是不可变集合，无法向其中添加或删除元素

补充：利用集合接口提供的of(T...)方法创建单元素集合，与此方法等效

集合接口提供的of(T...)方法，可创建任意个元素的集合

不可变集合

  • List<T> unmodifiableList(List<? extends T> list) 将可变的List封装成不可变的List
  • Map<K, V> unmodifiableMap(Map<? extends K, ? extends V> m) 将可变的Map封装成不可变的Map
  • Set<T> unmodifiableSet(Set<? extends T> set) 将可变的Set封装成不可变的Set

排序

  • Collections.sort(List list) 自然排序（升序）

  • Collections.sort(List list, Comparator c) 自定义排序

  • Collections.reverse(List list) 次序反转

  • Collections.swap(List list, int i, int j) 根据下标交换
  • Collections.shuffle(List list) 随机排序
  • Collections.rotate(List list, int distance) 将指定列表中的元素旋转指定的距离。当distance为正数时，将list后distance个元素整体移到前面。当distance为负数时，将 list的前distance个元素整体移到后面

查找

  • Collections.binarySearch(List list, Object key) 二分查找，传入的List必须有序

  • Collections.max(Collection coll) 根据自然排序，返回最大的元素

  • Collections.max(Collection coll, Comparator c) 根据自定义排序，返回最大的元素

  • Collections.min(Collection coll) 根据自然排序，返回最小的元素

  • Collections.min(Collection coll, Comparator c) 根据自定义排序，返回最小的元素

  • Collections.frequency(Collection c, Object o) 统计元素出现的次数

替换

  • Collections.replaceAll(List list, Object oldVal, Object newVal) 用新元素替换旧元素
  • Collections.fill(List list, Object obj) 用指定的元素替代集合中的所有元素

同步

  • List<T> synchronizedList(List<T> list) 将线程不安全的List变为线程安全的List

  • Map<K,V> synchronizedMap(Map<K,V> m) 将线程不安全的Map变为线程安全的Map

  • Set<T> synchronizedSet(Set<T> s) 将线程不安全的Set变为线程安全的Set

注意：从Java 5开始，引入了更高效的并发集合类，上述方法效率低下，不推荐使用
